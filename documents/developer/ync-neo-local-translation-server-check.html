ゆかコネNEO ローカル翻訳サーバー確認手順

Parapper の localhost 翻訳サーバーを、ゆかコネNEO の「Local Translation(Custom API)」から OpenAI Format で呼ぶための確認手順です。

対象: Windows Parapper v0.4 以降 既定ポート: 18081
重要: NEO 本体の入力 API である /api/input と、翻訳エンジンの Local Translation(Custom API) は別物です。 この確認では NEO の翻訳エンジン設定から Parapper の http://127.0.0.1:18081 に送ります。NEO 本体の既定ポート 15520 には送らないでください。

1. 役割の整理

通信先 用途 今回使うか
http://127.0.0.1:15520/api/input ゆかコネNEO 本体へ認識テキストを入力する API。 使いません。
http://127.0.0.1:18081/v1/chat/completions Parapper が受ける OpenAI chat completions 互換 endpoint です。 使います。NEO 側には base URL の http://127.0.0.1:18081 を設定します。

2. Parapper 側の事前設定

Parapper の MT タブで、翻訳マッピング一覧の上にある「localhost 翻訳サーバー」を設定します。

  1. mode を サーバーのみ にします。内部翻訳 ON 時だけ起動したい場合は 翻訳ON時 にします。
  2. 受信 port を 18081 にします。
  3. サーバーモデルを LFM2-350M-ENJP-MT-ONNX / ONNX Community Q4 にします。
  4. mode / port / model の変更後、Parapper を再起動します。

設定ファイルで確認する場合、通常の保存先は次の場所です。

%APPDATA%\com.parakeet-inc.parapper\config.json

ローカルサーバーだけを明示的に起動する場合は、設定 JSON のトップレベルに次の値を入れます。

{
  "translation_local_server_mode": "on",
  "translation_local_server_port": 18081,
  "translation_local_server_model": "lfm2_q4"
}

内部翻訳を ON にした時だけサーバーも起動する確認にしたい場合は、次の組み合わせにします。

{
  "translation_enabled": true,
  "translation_local_server_mode": "auto",
  "translation_local_server_port": 18081,
  "translation_local_server_model": "lfm2_q4"
}
項目 推奨値 意味
translation_local_server_mode on 内部翻訳 ON/OFF と独立して localhost 翻訳サーバーを起動します。
translation_local_server_port 18081 NEO 側、または確認コマンドから接続するポートです。
translation_local_server_model lfm2_q4 LiquidAI のモデルを変換した ONNX Community 配布のQ4ローカル翻訳モデルです。
モデル取得について: Hugging Face の ONNX Community repository からQ4モデルを取得します。base model は LiquidAI/LFM2-350M-ENJP-MT です。

3. ゆかコネNEO 側で見るべき設定

NEO の翻訳設定で、翻訳先言語のエンジンに ローカル翻訳（個人 / Custom API） を選びます。翻訳/発話連携サーバプラグインの設定画面を探す必要はありません。

  1. URL は http://127.0.0.1:18081 にします。
  2. POST mode は OpenAI Format を選びます。
  3. Model は LFM2-350M-ENJP-MT-ONNX など任意の表示名で構いません。実際に使うモデルは Parapper の MT タブにある「サーバーモデル」です。
  4. /api/input や 15520 は指定しません。
バージョン注意: ゆかコネNEO のバージョンによっては Custom API への送信に対応していない場合があります。 リクエストが送信されない場合は、利用中のバージョンが Local Translation(Custom API) の OpenAI Format に対応しているか確認してください。
考え方: 今回は「NEO から Parapper に翻訳を依頼する」構成です。 Parapper から NEO に認識テキストを送る通常の連携とは向きが逆になるため、ポート番号も 15520 ではなく 18081 を使います。

4. PowerShell で先に疎通確認する

NEO 側を触る前に、Parapper のローカル翻訳サーバーだけで確認します。Parapper を起動した状態で PowerShell を開き、次を実行します。

4.1 OpenAI Format の確認

$openai = "http://127.0.0.1:18081/v1/chat/completions"
Invoke-RestMethod -Method Post -Uri $openai -ContentType "application/json" -Body '{"model":"LFM2-350M-ENJP-MT-ONNX","messages":[{"role":"system","content":"Translate into English."},{"role":"user","content":"こんにちは。"}]}'

期待する結果は、object が chat.completion、choices[0].message.content に翻訳結果が入る JSON です。

5. 間違った接続先の確認

誤設定を検出するため、次の確認もしておくと切り分けが楽になります。

Invoke-RestMethod -Method Post -Uri "http://127.0.0.1:18081/api/input" -ContentType "application/json" -Body '{"model":"LFM2-350M-ENJP-MT-ONNX","messages":[{"role":"user","content":"こんにちは。"}]}'

この request は翻訳 request として扱われないのが正しい結果です。/api/input で翻訳が通る構成にしないでください。

6. 確認チェックリスト

結果 確認項目
□ OK Parapper の translation_local_server_mode が on、または auto かつ translation_enabled: true になっている。
□ OK Parapper 再起動後、POST http://127.0.0.1:18081/v1/chat/completions の疎通確認が成功する。
□ OK NEO 側の翻訳エンジンが ローカル翻訳（個人 / Custom API） で、URL が 127.0.0.1:18081 系になっている。
□ OK POST mode は OpenAI Format になっている。
□ OK NEO 側で /api/input や 15520 を翻訳サーバーとして指定していない。
□ OK OpenAI Format の request が /v1/chat/completions に届いている。
□ OK 実翻訳で object: chat.completion と choices[0].message.content が返る。

7. トラブルシュート

症状 確認すること
接続できない Parapper を再起動したか、translation_local_server_mode が off のままではないか、port が 18081 で一致しているかを確認します。
OpenAI Format の確認が error になる model file が存在するか、指定 model が lfm2_q4 になっているか、request text が空でないかを確認します。
NEO 側から翻訳が来ない NEO 側の翻訳エンジンが ローカル翻訳（個人 / Custom API） になっているか、URL が http://127.0.0.1:18081 になっているか、利用中のバージョンが OpenAI Format の送信に対応しているかを確認します。
別アプリの 18081 と競合する Parapper 側の translation_local_server_port と NEO 側の接続先 port を同じ別番号に変更し、Parapper を再起動します。
完了条件: PowerShell の OpenAI Format 確認が成功し、NEO 側からも /v1/chat/completions に対して翻訳 request が届けば、ローカル翻訳サーバー連携の確認は完了です。
