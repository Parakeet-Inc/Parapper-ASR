# 使い方

<!-- cspell:words parapper Silero ReazonSpeech sherpa OSCQuery Piper Supertonic Namo UNAS Kristin VOICEVOX VRSNS -->

Parapper の基本的な使い方と、各設定項目について説明します。

## 初回セットアップ

1. アプリを起動します。初回はオンボーディング画面が開きます。
2. `UIの使用言語 / UI language` で表示言語を選びます。
3. `設定プリセット` で最初に使うワークフローを選びます。
4. 「反映してモデルをダウンロード」を押します。選んだプリセットに必要なモデルが自動でダウンロードされます。
5. ダウンロードが完了したら、メイン画面で入力デバイス(マイク)を選びます。
6. 入力音量メーターを見ながらマイク入力音量を調整します。
7. 「スタート」を押すと認識が始まります。

> 日本語モデル(ReazonSpeech K2 v2)を `int8-fp32` でダウンロードする場合、VADと合わせて約170MBになります。回線状況によっては少し時間がかかります。

## 設定プリセット

用途に合わせて、必要な機能をまとめてONにする組み合わせです。プリセットを選んで「反映してモデルをダウンロード」を押すだけで、必要なモデルを取得し設定もまとめて切り替わります。

| プリセット | 主な用途 |
| --- | --- |
| **日本語文字起こし シンプル** | 一番軽い設定。日本語の文字起こしだけを行います。動作確認やお試しに。 |
| **日本語文字起こし** | NCとTurn Detectorを有効にした、配信向けの日本語文字起こしです。 |
| **日→英 翻訳** | 日本語で話した内容を英語に翻訳して字幕にします。海外向けの配信に。 |
| **日→英 翻訳読み上げ** | 上記に加えて、翻訳した英語を音声で読み上げます。海外視聴者に声でも届けたいときに。 |
| **日英⇔英日 翻訳読み上げ** | 日本語と英語を自動で切り替え、相互に翻訳と読み上げをします。多言語のVRChat会話などに。 |

自分用に調整した設定は「その他」タブから別名で保存できます。

## メイン画面

- **入力デバイス**: マイクを選びます。USBマイクやオーディオインターフェースもここから選択できます。
- **入力音量メーター**: マイク入力レベルが視覚化されます。話したときに緑〜黄色に振れる程度になるよう調整します。
- **スタート / ストップ**: 認識のON/OFFを切り替えます。
- **認識ログ**: リアルタイムの認識結果、翻訳結果がここに並びます。CSVで書き出すこともできます。

## 設定パネル

設定パネルは画面上から開き、用途別に複数のタブに分かれています。順番に説明します。

### 接続

ゆかコネNEOや他のアプリとの連携に関する設定です。

- **ゆかコネNEOへ送信**: 認識した文字をゆかコネNEOに送ります。配信字幕やOBS連携に使います。
- **ゆかコネNEO HTTP API ポート**: ゆかコネNEO本体との接続先ポート番号です。通常は既定値(`15520`)のままで使えます。「探す」ボタンで自動検出することもできます。
- **翻訳/発話プラグイン HTTP ポート**: ゆかコネNEOの「翻訳/発話連携サーバプラグイン」との接続先ポート番号です。通常は既定値(`8080`)のままで使えます。
- **ゆかコネNEO送信タイミング**:
  - `ASR途中も送る`: 認識中の途中経過も送ります。リアルタイム感が出ます。
  - `確定後に送る`: 発話が完了してから送ります。途中で文字が変わるのが気になるときに。
- **VRChatでミュートの時は送信しない**: VRChatでマイクをミュートにしている間、字幕の送信を自動で止めます。
- **モデルの状態**: 必要なモデルが揃っているかをタブごとに表示します。「ダウンロード」を押すと不足分を取得できます。

> ゆかコネNEO連携を使う場合、ゆかコネNEO本体が **v2.3.58 以降** であることが必要です。

### NC(ノイズキャンセリング)

マイク音声のノイズを抑制してから認識にかける機能です。

- **ノイズキャンセリングを有効化**: ONにするとマイク音声のノイズを抑えます。キーボード音や空調音がうるさい環境におすすめです。CPU使用量は少し増えます。
- **NCモデル**: ノイズキャンセリングに使うモデルを選びます。

### VAD / Turn Detector

「いつ話し始めて、いつ話し終わったか」を判定するための設定です。ここの設定で字幕の出やすさやテンポが大きく変わります。

- **Turn Detector(発話完了の判定方法)**:
  - `Simple`: 一定の無音時間で発話完了と判定します。軽くて速い動作です。
  - `Namo`: AIモデルで「文として終わったか」を判定します。間が空いても文の途中なら継続として扱われるので、字幕が途中で切れにくくなります。ASRで選んだ言語に合わせて、日本語・英語・多言語のモデルが自動で使い分けられます。
- **途中経過を表示する**: ONにすると、話している途中でも認識結果がリアルタイムに字幕として更新されます。OFFだと発話が確定してから一気に表示されます。
- **途中表示までの無音時間**: 途中経過を更新するまでに必要な無音の長さ(ミリ秒)。短くするほどテンポが速くなります。
- **完了までの無音時間**: 発話が完了したと判定するまでに必要な無音の長さ(ミリ秒)。話の間の取り方に合わせて調整します。
- **発話判定までの時間**: 音が聞こえてから「発話が始まった」と判定するまでの時間(ミリ秒)。
- **VADしきい値**: 「音声か無音か」を判定する感度。小さい声も拾いたいなら下げ、ノイズを拾いやすければ上げます。
- **Namo関連設定(Turn DetectorがNamoのとき)**: 判定の厳しさや、参照する直前テキストの長さなどを調整できます。基本は既定値のままで問題ありません。

### ASR(音声認識)

音声認識エンジン自体の設定です。

- **ASRモデル**: 認識する言語に合わせて選びます。
  - 日本語(ReazonSpeech K2 v2)
  - 英語(NeMo Parakeet TDT 0.6B v2)
  - 多言語(NeMo Parakeet TDT 0.6B v3、英語を含むヨーロッパ系25言語)
- **量子化精度(日本語のみ)**: 認識の精度と速度のバランスを `int8` / `int8-fp32` / `float32` の3段階から選びます。既定の `int8-fp32` がバランス型です。
- **CPUスレッド数**: 音声認識に使うCPUコア数です。`max` で自動。配信ソフトと共存するなら数を絞ると他のソフトに余裕が出ます。
- **入力前に音量を正規化する**: マイク入力の音量レベルを揃えてから認識します。マイクの音量が安定しないときに有効です。
- **多言語ASRを使い分ける**: ONにすると、発話の言語を自動判定して使うASRモデルを切り替えます。日本語と英語が混ざる場面で便利です。
- **使用するASRモデル**: 多言語使用時に、自動切り替えの候補に入れるモデルを選びます。

### MT(翻訳)

認識結果を別の言語に翻訳する機能です。翻訳にはゆかコネNEOの翻訳プラグインを利用します。

- **翻訳を有効化**: ONにすると認識結果を翻訳します。
- **翻訳タイミング**: 認識途中で翻訳を始めるか、発話が確定してから翻訳するかを選びます。
- **翻訳マッピング**: 「どのASRモデルの認識結果を、何語に翻訳するか」を組み合わせで設定します。例えば「日本語ASRの結果は英語に翻訳」「英語ASRの結果は日本語に翻訳」のように複数登録できます。

### TTS(読み上げ)

認識テキストや翻訳テキストを音声で読み上げる機能です。

- **読み上げマッピング**: 「どのテキストを、どの読み上げ先で、どの話者で読むか」の組み合わせを設定します。
  - **ソース**: 認識結果(ASR)か、翻訳結果かを選びます。
  - **対象**: 特定の言語・モデルにだけ適用することもできます。
  - **読み上げ先**: ゆかコネNEO、またはローカルTTSから選びます。
  - **話者**:
    - ゆかコネNEOの場合: ゆかコネNEOに登録されているVOICEVOXなどの話者から選びます。「話者一覧を取得」ボタンで最新の一覧を取得できます。
    - ローカルTTSの場合: 以下のエンジンから言語と話者を選びます。
      - Piper voices: 英語のみ(Kristin / John / Norman)
      - Supertonic 2: 英語・韓国語・スペイン語・ポルトガル語・フランス語
      - Supertonic 3: 上記に加えて日本語など、計23言語に対応
  - **ミュート**: 一時的にこのマッピングだけ無効にできます。
- **出力デバイス(ローカルTTS時)**: 読み上げ音声を出すスピーカー/仮想オーディオデバイスを選べます。
- **音量**: 読み上げ音量を `-20dB` 〜 `+20dB` の範囲で調整します。

> ゆかコネNEO側の本文読み上げと、Parapper側の翻訳読み上げを同時にONにすると、ゆかコネNEOの読み上げキューが混み合って遅延することがあります。どちらかに寄せるのがおすすめです。

### その他

便利機能とメンテナンス用の設定です。

- **設定プリセットの保存・反映・削除**: 現在の設定をプリセットとして保存できます。配信内容ごとに保存しておくと、ワンタッチで切り替えられます。デフォルトプリセットは削除できません。
- **ASR入力音声の再生デバッグ**: ONにすると、ASRに渡した音声を後から再生して確認できます。「思っていた認識結果と違う」ときの原因確認に便利です。メモリ使用量が増えるので、必要なときだけONにすることをおすすめします。
- **ASRログの保持件数**: 画面に表示する認識ログの上限件数です。長時間配信ではある程度の上限を設けたほうがメモリにやさしいです。
- **Debug音声の保持件数**: 上記のデバッグ用音声を何件まで残すかの上限です。
- **設定をリセットする**: すべての設定を初期状態に戻します。認識を停止してから実行してください。

### ライセンス

アプリで使用しているAIモデルや、各種ライブラリのライセンス一覧を表示します。動画や配信での利用時、クレジット表記の確認にも使えます。

## ゆかコネNEOとの連携

「ゆかコネNEOへ送信」をONにすると、認識した文字をゆかコネNEOに送ります。

- ポート番号は通常 `15520` で、本体側の設定に合わせて変更できます。
- 翻訳・読み上げを使う場合、ゆかコネNEOで **翻訳/発話連携サーバプラグイン** を有効にしておく必要があります(通常 `8080`)。
- ゆかコネNEOは **v2.3.58 以降** の安定版が必要です。古いバージョンを使っている場合は、先にアップデートしてください。

## VRChatとの連携

「VRChatでミュートの時は送信しない」をONにすると、認識開始時にVRChatからミュート状態を読み取り、ミュート中は字幕送信を自動でスキップします。

- VRChat側で **OSC** を有効にしておく必要があります(VRChatのRadial Menu → Options → OSC → Enabled)。
- Parapperとは別のアプリでVRChatのOSCを使っている場合は、競合しないよう設定を見直してください。

## トラブルシューティング

- **モデルダウンロードが進まない**: 「接続」タブの「モデルをダウンロード」を再度押してみてください。ファイアウォールや回線環境も確認します。
- **字幕がなかなか出ない**: 「VAD / Turn Detector」の `完了までの無音時間` を短くする、Turn Detectorを `Simple` に変える、`途中経過を表示する` をONにするなどを試してください。
- **発話が途中で切れて変な区切りになる**: Turn Detectorを `Namo` に変えると改善することがあります。
- **CPU使用率が高い**: 「ASR」タブで `CPUスレッド数` を絞る、NCをOFFにする、`量子化精度` を `int8` にするなどを試してください。
- **認識結果がおかしい**: 「その他」タブで `ASR入力音声の再生デバッグ` をONにすると、実際にASRに入っている音声を確認できます。マイク音量・ノイズ・正規化設定を見直してください。
- **読み上げが遅れる**: ゆかコネNEO側の本文読み上げと、Parapper側の翻訳読み上げが同時に動いていないか確認してください。

## ライセンス表示

アプリ内の「設定 > ライセンス」タブから、使用しているモデルおよび依存ライブラリのライセンス一覧を確認できます。
